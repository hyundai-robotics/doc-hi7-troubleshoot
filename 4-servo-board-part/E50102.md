# E50102. (O축) 엔코더 수신 데이터 길이 이상

## 1. 개요

서보 안전 보드는 서보 모터 제어를 위해 엔코더와 시리얼 통신을 수행하며, 주기적으로 엔코더 데이터를 수신합니다.<br>
해당 에러는 엔코더로부터 수신된 데이터의 길이가 규정치에 부합하지 않을 경우 발생합니다.<br>

엔코더로부터 수신된 데이터 길이가 맞지 않는 경우는 주로 배선 불량 또는 엔코더 쉴드 라인 처리 문제로 인해 엔코더 신호선에 노이즈가 유입될 때 발생할 수 있습니다.

## 2. 원인 및 점검

{% hint style="info" %}

(1)	엔코더 배선을 점검하십시오.<br>
(2) 모터(엔코더)를 교체 시험하십시오.<br>
(3)	서보안전 보드를 교체 시험하십시오.<br>
(4)	조치 완료후 배선의 통신상태를 점검하십시오.<br>

{% endhint %}

(1)	엔코더 배선을 점검하십시오.<br>
엔코더의 배선점검 순서는 다음과 같습니다.<br>
1차: 엔코더 배선에 관련된 커넥터들의 접촉 불량여부를 점검하십시오.<br>
2차: 엔코더 배선의 단락 유무를 점검하십시오. 멀티미터(테스터기)와 같은 장비를 이용하여 각 상의 배선을 1:1로 체크하십시오.<br>
3차: 엔코더 배선을 교체시험 하십시오.<br>

엔코더 배선이 단선되지 않고 쉴드선의 접촉불량, 엔코더 신호선과 타 전력선 또는 로봇 본체 금속부위와의 접촉 등의 현상이 있을 경우에는 단락유무 검사로는 검사할 수 없으므로 배선교체 시험을 하여주십시오.

* 제어기 내부 배선을 점검하십시오.<br>
CNEN13,46(BD642)커넥터와 CEC1간의 배선을 점검하십시오.

* 제어기와 로봇간의 배선을 점검하십시오.<br>
Hi7-N제어기의 경우, CEC1와 CER1간의 배선을 점검하십시오.

![](../_assets/4.서보보드/hi7_robot_encoder_cable.png)<br>
그림 4.5.2 Hi7-N 제어기와 로봇간 기본 설치 구성도

그림 4.5.3 Hi7-N 제어기와 로봇간 기본 설치 구성도 상세

* 본체 내부의 배선을 점검하십시오.<br>
CER1과 엔코더측 커넥터간의 배선을 점검하십시오. 배선점검은 로봇 보수설명서의 배선 접속도를 참고하시기 바랍니다.

![](../_assets/4.서보보드/robot_inner_cable.png)<br>

(2)	모터(엔코더)를 교체 시험하십시오.<br>
서보 모터를 교체한 후 에러가 발생하지 않으면 서보모터의 불량입니다. 서보 모터를 정상품으로 교체하여 주십시오. 아래 그림은 HS165 로봇의 각 축 모터의 위치를 나타내고 있으며, 다른 로봇은 해당 기구 보수설명서를 참고하여 교체하시기 바랍니다.

![](../_assets/4.서보보드/HS165_motor_position.png)<br>
그림 4.5.5 HS165 로봇의 각 축 모터 위치

(3)	서보안전 보드를 교체 시험하십시오.<br>
서보안전 보드를 교체한 후 에러가 발생하지 않으면 서보안전 보드의 불량입니다. 서보안전 보드를 정상품으로 교체하여 주십시오.

![](../_assets/4.서보보드/hi7_controller_bd642.png)<br>
그림 4.5.6 N제어기 서보안전 보드 교체

(4)	조치 완료후 배선의 통신상태를 점검하십시오.<br>
문제부분의 조치가 완료된후 『엔코더 통신실패 횟수 표시 기능 메뉴얼』을 참고하여 통신상태를 점검 하십시오.
