# E2301. CPUERR 신호 불일치 (H6COM 테스크 에러)

## 1. 개요

CPUERR 신호 불일치 (H6COM 테스크 에러) 이상이 발생하였습니다. H6COM-T의 일부 태스크의 수행시간이나 수행주기가 정상범위를 넘어간 경우 해당알람이 발생할 수 있습니다. 이 경우 시스템은 모터온 할 수 없습니다.

## 2. 원인 및 점검


{% hint style="info" %}

(1)	메인의 테스크 주기가 정상 범주 내에 있지 않은 경우
* 최신 버전의 SW가 설치 되지 않은 경우
* 외부 장비와의 통신이상이 시스템에 영향을 주는 경우

(2)	안전보드(BD632)에 이상이 있는 경우
* 안전보드(BD632)를 점검하는 방법

{% endhint %}


### (1)	메인의 테스크 주기가 정상 범주 내에 있지 않은 경우

#### [최신 버전의 SW가 설치 되지 않은 경우]

TP화면에서 서비스 -> 시스템 진단 -> 시스템 버전 으로 이동하여 현재 설치된 버전이 최신 버전의 소프트웨어인지 확인한다.

#### [외부 장비와의 통신이상이 시스템에 영향을 주는 경우]

외부 장비와의 통신 활성화 유무에 따라 시스템에 영향을 주는 경우가 생길 수 있습니다. 이 경우 외부 장비와의 통신 활성화 유무에 따라 /tIOMain0의 태스크 실행시간을 비교하여 주십시오.

만약 최대수행시간이 10000 usec를 초과하는 경우 소프트웨어 개발팀과 사용조건에 대해 상의 하십시오.

#### [태스크 실행시간 확인 방법]
창조정 -> 분할(상하) -> 아래 창 누르고 -> 선택 버튼 -> 태스크모니터 선택

### (2)	안전보드(BD632)에 이상이 있는 경우

#### [ 안전보드(BD632)를 점검하는 방법]

 ![](../_assets/3-Safety-io/E02301/그림9.png)<br>

1)	IO 전원 상태를 확인하는 방법<br>
A.	위 그림의 두개의 LED가 녹색 점등 상태 인지를 확인<br>
B.	IO 전원 LED가 적색 또는 소등인 경우 표시된 퓨즈가 정상 상태인지를 확인한다.<br>
C.	퓨즈가 끊겼다면 퓨즈를 교체하여 준다.<br>

2)	모터온시 IO전원이 불안정한 상태인지 확인<br>
A.	모터온시에 IO 전원 LED가 녹색 점등 상태 인지를 확인한다. <br>
B.	모터온 순간 적색으로 변하거나 소등되다면 모터온시 IO 전원이 불안정한 상태 임<br>

3)	IO 전원 상태가 불안정한 경우<br>
A.	IO 전원 커넥터 체결 상태 확인<br>
B.	IO 전원 케이블 점검<br>
C.	안전보드(BD632) 접지 상태 점검 (접지 케이블, 접지 단자 체결 상태)<br>
